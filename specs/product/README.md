# Product Overview

> Eatery is an offline-first restaurant POS system for independent restaurants,
> cafes, and quick-service outlets. It runs on commodity tablets, phones, and
> desktops — no proprietary hardware required.

---

## Target Market

- **Primary:** Single-location independent restaurants (1–30 tables)
- **Secondary:** Multi-location chains (cloud replication for consolidated reporting)
- **Tertiary:** Food trucks, pop-ups, ghost kitchens (mobile-first POS)

## User Personas

### Admin / Owner
Runs the business. Needs sales reports, menu management, staff oversight.
Uses a tablet or desktop at the counter.

### Waiter / Server
Takes orders at the table. Needs a fast, intuitive menu browser.
Uses a phone or small tablet.

### Kitchen Staff
Sees incoming orders on a KDS (Kitchen Display System).
Needs a clear, scroll-free feed grouped by station.

### Customer (future)
Browses the menu online, places orders for pickup/delivery.
Interacts through a web portal or ordering kiosk.

## Roadmap

| Phase | Focus | Key Features |
|-------|-------|-------------|
| **v1.0 (current)** | Single-location POS | Menu mgmt, order entry, cart, payment logging, basic reports, Bluetooth printing |
| **v1.5** | Multi-device sync | OpLog protocol, device discovery, host election, offline resilience |
| **v2.0** | Kitchen display | Real-time KDS with station routing, bump bars, prep time tracking |
| **v2.5** | Cloud replication | Firebase/Supabase sync target, cross-location dashboard, remote monitoring |
| **v3.0** | Online ordering | Customer-facing menu portal, pickup/deliver orders, payment gateway |
| **v3.5** | Compliance & scale | Z-reports, tax reconciliation, inventory management, multi-currency |
