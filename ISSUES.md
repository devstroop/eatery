| **LOW** | Cosmetic, dead code, or minor UX polish | Fix when convenient |

## 8. Sync Layer

### FIXED (since audit)

| Old ID | Issue | Fixed In |
|--------|-------|----------|
| **S1** | `_sendMessage()` TODO stub | Audit was stale. `SyncService` doesn't have `_sendMessage`. Sync uses `SyncServer`/`SyncClient` with real WebSocket channels. |
| **S2** | OpLog never written | `notifyMutation()` is now called from all SQLite repositories (21 call sites). Flows through MutationHook → SyncCoordinator → OpLogService. |
| **S3** | `_getUptimeSeconds()` returns clock | `sync_service.dart` no longer has this method. Host election logic was removed in refactoring. |
| **S7** | Graceful role transition missing | Sync status chip + settings sheet exist. mDNS auto-discovery added (605ef55). |

### REMAINING

| ID | Issue | File | Details |
|----|-------|------|---------|
| **S4** | Clock validation missing | `op_log_service.dart:96-98` | `advanceClockTo()` advances unconditionally without validating source. Malicious peer could set incorrect clock. |
| **S5** | Full snapshots in OpLog | `op_log_entry.dart` | Full entity snapshots in `data`. Storage grows quickly for 1000+ orders/day. Periodic compaction needed. |
| **S6** | No device identity management | `sync_service.dart` | `deviceId` is a parameter. No persistence across restarts, no user-facing name. |
| **SR1** | Sync `_applyEntry()` builds SQL from column names | `sync_coordinator.dart:184-196` | Dynamic SQL: `'INSERT OR REPLACE INTO $table (id, $columns) VALUES ...'`. EntityType is allowlisted but column names are not. Malicious peer could inject arbitrary column names. |
| **SR2** | OpLog IDs not proper UUIDs | `op_log_service.dart` | `'${deviceId}_$_clock'` — not a UUID v7 as comments suggest. Collision prone if device restarts and clock resets. |
| **SR3** | Sync entity allowlist incomplete | `sync_coordinator.dart:139-154` | Missing: `auto_print`, `kds_station`, `compliance_report`, `void_log_entry`. These tables exist in schema but can't be synced. |
